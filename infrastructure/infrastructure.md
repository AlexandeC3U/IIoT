### When Do You Need an External Load Balancer?

| Scenario | Load Balancer Needed? | Solution |
|----------|----------------------|----------|
| All devices inside cluster | âŒ No | ClusterIP (default) |
| External MQTT clients | âœ… Yes | LoadBalancer or NodePort service |
| Dashboard access | Maybe | Ingress (K3s has Traefik built-in) |
| External API access | Maybe | Ingress |

### External Load Balancer (If Needed)

For external MQTT access, add this to `services/emqx/service.yaml`:

```yaml
apiVersion: v1
kind: Service
metadata:
  name: emqx-external
  namespace: nexus
spec:
  type: LoadBalancer  # or NodePort
  selector:
    app.kubernetes.io/name: emqx
  ports:
    - name: mqtt
      port: 1883
      targetPort: 1883
    - name: mqtts
      port: 8883
      targetPort: 8883
```

For K3s without a cloud provider, use **MetalLB** or **NodePort**.

## Deployment Guide

### Prerequisites

| Requirement | Development | Production |
|-------------|-------------|------------|
| **Kubernetes** | Docker Desktop, minikube, kind | K3s, K8s (AKS/EKS/GKE) |
| **kubectl** | v1.25+ | v1.25+ |
| **Storage** | Default StorageClass | SSD-backed StorageClass |
| **Memory** | 8GB+ | 16GB+ |
| **CPU** | 4 cores | 8+ cores |

### Step-by-Step Deployment

```bash
# 1. Clone repository
git clone https://github.com/your-org/nexus-edge.git
cd nexus-edge

# 2. Install K3s (if using K3s)
curl -sfL https://get.k3s.io | sh -
export KUBECONFIG=/etc/rancher/k3s/k3s.yaml

# 3. Deploy to development
kubectl apply -k infrastructure/k8s/overlays/dev

# 4. Wait for pods to be ready
kubectl wait --for=condition=ready pod -l app.kubernetes.io/part-of=nexus-edge -n nexus --timeout=300s

# 5. Verify deployment
kubectl get pods -n nexus
kubectl get svc -n nexus
kubectl get pvc -n nexus

# 6. Check logs
kubectl logs -n nexus -l app.kubernetes.io/name=protocol-gateway -f
kubectl logs -n nexus -l app.kubernetes.io/name=data-ingestion -f

# 7. Access services (port-forward for development)
kubectl port-forward -n nexus svc/emqx 18083:18083 &
kubectl port-forward -n nexus svc/timescaledb 5432:5432 &

# Open EMQX Dashboard: http://localhost:18083 (admin/public)
# Connect to TimescaleDB: psql -h localhost -U postgres -d nexus_historian
```

### Production Checklist

- [ ] Update secrets with production values
- [ ] Configure external LoadBalancer or Ingress for MQTT
- [ ] Set up monitoring (Prometheus + Grafana)
- [ ] Configure backup for TimescaleDB PVC
- [ ] Set up alerting (PagerDuty, Slack)
- [ ] Enable TLS for MQTT and HTTP endpoints
- [ ] Configure resource limits based on actual usage
- [ ] Set up log aggregation (Loki, ELK)

## Summary

| Aspect | Development | Production |
|--------|-------------|------------|
| **Orchestration** | Docker Compose | Kubernetes (K3s) |
| **Replicas** | 1 each | 2-10 (HPA) |
| **Storage** | Docker volumes | PVCs (SSD) |
| **Networking** | Port mapping | ClusterIP + optional LB |
| **Scaling** | Manual | Automatic (HPA) |
| **HA** | No | Yes (multi-replica) |
| **Secrets** | .env files | K8s Secrets + Vault |
