Quick Start Options

Recommended

Option 1: Docker Compose

Start the entire platform with one command. Best for evaluation and development.

# Navigate to docker directory
cd infrastructure/docker

# Start all services
docker-compose up -d

# Start with monitoring (Prometheus + Grafana)
docker-compose --profile monitoring up -d

# View logs
docker-compose logs -f

Services Started:

Web UI localhost:8080 -
Gateway Core API localhost:3001 -
Protocol Gateway localhost:8085 -
Data Ingestion localhost:7000 -
EMQX Dashboard localhost:18083 admin / public
TimescaleDB localhost:5432 nexus / nexus_historian_secret
PostgreSQL (Config) localhost:5433 nexus / nexus_config_secret

With --profile monitoring:

Prometheus localhost:9090 -
Grafana localhost:3000 admin / admin

Option 2: Make Commands

Use the Makefile for common operations. Requires Make installed.

# Install dependencies
make install

# Start development environment
make dev

# Run tests
make test

# Build all services
make build

# View all commands
make help

Advanced

Option 3: Kubernetes

Deploy to a Kubernetes cluster. Requires kubectl and a running cluster.

# Development deployment
kubectl apply -k infrastructure/k8s/overlays/dev

# Production deployment
kubectl apply -k infrastructure/k8s/overlays/prod

# Check status
kubectl get pods -n nexus-system

# View logs
kubectl logs -f -n nexus-system \
  -l app.kubernetes.io/part-of=nexus-edge

Step-by-Step: Docker Compose

1Clone the Repository

git clone https://github.com/your-org/nexus-edge.git
cd nexus-edge

2Configure Environment (Optional)

# Copy the template
cp infrastructure/docker/env.template infrastructure/docker/.env

# Edit as needed (defaults work for local dev)
nano infrastructure/docker/.env

3Start Services

cd infrastructure/docker
docker-compose up -d
Tip: First run downloads images (~2GB). Subsequent starts are instant.

4Verify Services

# Check all containers are running
docker-compose ps

# Expected output:
NAME                STATUS
emqx                Up (healthy)
protocol-gateway    Up (healthy)
data-ingestion      Up (healthy)
timescaledb         Up (healthy)

5Access Dashboards

  • Web UI: http://localhost:8080
  • Gateway Core API: http://localhost:3001/health
  • Protocol Gateway API: http://localhost:8085/health
  • Protocol Gateway Metrics: http://localhost:8085/metrics
  • Data Ingestion Health: http://localhost:7000/health
  • EMQX Dashboard: http://localhost:18083 (admin / public)

Common Operations

Stop Services

cd infrastructure/docker
docker-compose down

View Logs

# All services
docker-compose logs -f

# Specific service
docker-compose logs -f protocol-gateway

Restart a Service

docker-compose restart protocol-gateway

Clean Up (Remove Data)

# Stop and remove volumes (deletes all data!)
docker-compose down -v
Warning: This deletes all stored data including TimescaleDB history.

Development Workflow

Run Protocol Gateway Locally (with hot reload)

# Terminal 1: Start infrastructure
cd infrastructure/docker
docker-compose up -d emqx timescaledb

# Terminal 2: Run gateway with air (hot reload)
cd services/protocol-gateway
air

Build Docker Images Locally

# Build all images
make docker-build

# Or build individually
docker build -t nexus/protocol-gateway:dev services/protocol-gateway
docker build -t nexus/data-ingestion:dev services/data-ingestion

Troubleshooting

Container won't start

# Check logs for errors
docker-compose logs protocol-gateway

# Check if ports are in use
netstat -an | grep 8080

Connection refused to EMQX

# Wait for EMQX to be healthy
docker-compose ps

# Check EMQX logs
docker-compose logs emqx

Reset everything

# Nuclear option - removes all containers and data
docker-compose down -v --remove-orphans
docker system prune -f
docker-compose up -d
